# 使用者實機測試流程（v3 + v4）

> 給第一次拿到專案的人，從零跑完整個 v3「逐行真人錄音」+ v4「多劇本 / 匯入」流程。
> 含每個階段預期看到的 UI 反饋；建議用桌機 + Chrome（錄音可改用手機）。

## 0. 前置需求

- Node.js 20+ 與 npm
- 桌機 Chrome / Edge（其他現代瀏覽器亦可，Firefox 無 STT）
- 任一支援 MediaRecorder 的裝置做錄音（桌機 / 手機 / 平板皆可）
- 約 15 分鐘空檔

## 1. 安裝與啟動（< 2 分鐘）

```bash
cd "Script rehearsal"
npm install         # 約 30 秒（無 Whisper 依賴後變很快）
npm run dev
# 開啟 http://localhost:3000
```

預期看到（**v6 / M28 起首開為空狀態**）：
- 首頁標題「劇本對練平台」
- 引導文案「尚未匯入任何劇本。請先匯入劇本以開始對練。」
- 主 CTA「匯入劇本」
- **不應出現**任何角色選擇器、範圍 / 提示模式、ScriptSwitcher、AudioManager

## Step 0：空狀態（首開 / 清空 IDB 後，< 1 分鐘）

驗證 v6 / M28「無預設劇本」行為。

1. （首次跑）直接開 `http://localhost:3000`；或：
   - DevTools → Application → IndexedDB → 刪除 `script-rehearsal-audio` DB
   - 重新整理頁面
2. 預期看到：標題 + 引導文案 + 「匯入劇本」CTA，**僅此而已**
3. 點「匯入劇本」→ 跳轉到 `/scripts/import`
4. 完成至少一份劇本的匯入（最簡單：純文字 tab 貼幾行 `角色：台詞`）
5. 儲存後回首頁 → 立即看到正常 UI（劇本面板 / 角色 / 範圍 / 提示 / 開始對練）

**通關條件**：清空 IDB 後一律落到空狀態；匯入後 `subscribeActiveScriptId` 觸發首頁自動切到正常 UI，無需手動重新整理。

## Step 1：v1.0 純 TTS 回歸（不錄音，< 3 分鐘）

驗證在「未錄任何音檔」狀態下，產品行為與 v1.0 完全一致。

1. 直接選角色「維克多」
2. 範圍選「第 41 頁」
3. 提示模式選「完整」
4. 點「開始對練」
5. 預期：所有對手台詞由瀏覽器 TTS 唸出；StatusBar 顯示來源「TTS」
6. 輪到維克多時系統暫停；念完按空白鍵或讓 STT 推進
7. 走完該頁 → 「練習完成 +1」彈窗

**通關條件**：StatusBar 從頭到尾都顯示「TTS」徽章，未顯示「真人錄音」。

## Step 2：錄音頁逐段錄、確認、跳行（5 分鐘）

1. 回首頁；找到「逐行錄音」區塊
2. 在「維克多」這列點「錄音」→ 跳轉到 `/record/維`（或 URL encoded）
3. 預期看到：
   - Header：「維克多 的錄音 ｜ 進度 0 / N」
   - 中央顯示第 1 行台詞（含 page / globalIndex 標籤）
   - 「⏺ 點擊開始錄音」按鈕
4. 第一次點時會跳出麥克風權限請求，允許
5. 點「⏺ 開始錄音」→ 唸完該行 → 點「⏹ 停止」
6. 進入 preview 狀態：可試聽、可「🔁 重錄」或「✓ 確認下一行」
7. 點「✓ 確認下一行」→ 預期：
   - Header 進度變 `1 / N`
   - 自動前進到第 2 行
   - 下方迷你地圖該位置打 ✓
8. 連續錄 3-4 行，驗證流程順暢
9. 中間嘗試點「◀ 上一行」/「下一行 ▶」測試跳轉
10. 在 preview 狀態下點跳行會跳 confirm「將丟棄這次錄音」

**通關條件**：可連續確認多行、跳行有 confirm 守衛、進度同步更新。

## Step 3：半錄 + 關頁 + 重開續錄（< 2 分鐘）

驗證 IndexedDB 持久化正確。

1. 在 Step 2 已錄 3-4 行的狀態下，直接關閉分頁（或按「返回設定」）
2. 重新開啟 `http://localhost:3000`
3. 設定頁「逐行錄音」區塊應顯示維克多進度 `3-4 / N`（與剛剛一致）
4. 再次點「錄音」進入 `/record/維`
5. 預期：
   - 已錄過的行在迷你地圖標 ✓
   - 游標停在第 1 行（可手動跳到下一未錄行繼續錄）

**通關條件**：關頁後計數正確、迷你地圖正確標記、可續錄。

## Step 4：全錄完 → 對練全程真人聲 + StatusBar 徽章（5 分鐘）

1. 把維克多剩下的行全錄完，進度跑到 `N / N`
2. 回首頁
3. **選非維克多的角色**（例如選「胡利安」自己念，對手就會有維克多的台詞）
4. 範圍選「第 41 頁」、提示模式選「完整」
5. 點「開始對練」
6. 走戲時注意：
   - 維克多的台詞會播放剛才錄的真人錄音
   - 其他未錄的角色（娜塔莉亞 / 卡蘿莉娜）仍走 TTS
   - StatusBar 在播放真人錄音時顯示「真人錄音」徽章；TTS 行顯示「TTS」徽章

**通關條件**：維克多每一行都播真人錄音、StatusBar 徽章正確切換。

## Step 5：重錄單行（< 1 分鐘）

1. 回首頁 → 點「錄音」進入維克多的錄音頁
2. 用迷你地圖點到任一已錄行（綠色 ✓）
3. 上方顯示「✓ 已錄」標記，但仍可重新錄
4. 點「⏺ 開始錄音」→ 重新錄 → 確認下一行
5. 對練時該行播放新版錄音

**通關條件**：可覆蓋舊片段；對練時播新版。

## Step 6：改劇本 → 橘色徽章 → 重錄缺漏行（3 分鐘）

驗證 scriptHash 機制。

1. 在首頁的「劇本」面板點當前劇本的「編輯」進入 `/scripts/[id]/edit`
2. 改動維克多任何一行的內容（例如加個字）後點儲存
3. 回首頁
4. 預期「逐行錄音」區塊維克多列出現**橘色「劇本已變更」徽章**
5. 進對練頁仍可走，被動補的行（已錄但內容變動）依舊播放舊錄音
6. 進錄音頁，找到變動的那一行，重錄之
7. 全角色都重錄後，徽章消失（所有片段 hash 都對齊新劇本）

**通關條件**：橘色徽章正確出現；重錄完所有片段後徽章消失。

> 加碼驗證：若把所有維克多片段都刪光（Step 7），徽章也會消失（無錄音則無變更概念）。

## Step 7：刪除全部 → 回 v1.0（< 1 分鐘）

1. 設定頁 → 維克多列點「刪除全部」（或在角色卡操作）
2. 跳 confirm 「將刪除維克多所有錄音片段」→ 確認
3. 進度回到 `0 / N`、徽章消失
4. 開始對練 → 維克多的台詞全部走 TTS（StatusBar 顯示「TTS」徽章）

**通關條件**：刪除後完全回歸 v1.0 純 TTS 行為，無任何殘留。

## v4 補測：多劇本與匯入（M17–M22）

> 完成上述 v3 七步後，繼續跑下面三步驗證 v4 行為。

### Step 8：匯入純文字劇本 + 編輯（3 分鐘）

1. 首頁右上點「匯入新劇本（文字 / PDF / 圖片）」→ `/scripts/import`
2. 在「純文字」tab 貼入幾段對白（格式 `角色：台詞`，連空行視為換頁）
3. 解析完跳到編輯頁，左側角色面板、中央多頁、行內可改 character / text / 上下移
4. 改一兩行後點「儲存」→ 回首頁
5. 首頁「劇本」面板出現新劇本（自動 active）；ScriptSwitcher 顯示 2 份

**通關條件**：可在站內維護兩份劇本、切換流暢。

### Step 9：多劇本錄音隔離（4 分鐘）

驗證 M22 三段複合 key 不串音。前置：手動匯入兩份不同的劇本（劇本 A、劇本 B），確保兩者都有同一個 character key（例如「維」）。

1. 在劇本 A 的「維」角色錄個 1–2 行
2. 切到劇本 B，進「維」的錄音頁
3. **預期**：劇本 B 的「維」進度為 `0 / N`，**不顯示**劇本 A 的錄音
4. 在劇本 B 的「維」錄一行；切回劇本 A
5. **預期**：劇本 A 的「維」進度仍為原本錄的 1–2 行（劇本 B 新錄的不會出現）
6. DevTools → IndexedDB → `script-rehearsal-audio` → `audioSegments` → 看到 record 含 `scriptId` 欄位

**通關條件**：兩份劇本同名角色完全獨立、彼此看不到對方的錄音。

### Step 10：IndexedDB schema 升級回歸（< 2 分鐘）

驗證 M22 v4 → v5 migration 正確。

1. 在已升級 v5 的環境下：DevTools → IndexedDB → 確認 `audioSegments` 的 keyPath 為 `["scriptId", "characterKey", "globalIndex"]`
2. 從乾淨環境（清掉 IDB）跑 `npm run dev` → 首頁落到空狀態（Step 0），匯入一份劇本後可順利錄音
3. 若曾在 v3/v4 環境錄過音（手動 downgrade `DB_VERSION` 測，或保留歷史機器升級）：升級後 `audioSegments` 的舊 record 全部歸為 `scriptId: 'default'`；v6 起無自動 seed，需先匯入劇本後手動 `setActiveScriptId('default')`（或在 ScriptSwitcher 切換）才會看到舊錄音

**通關條件**：v0 → v5、v3 → v5、v4 → v5 皆可順利開啟資料庫；舊錄音不丟失。
