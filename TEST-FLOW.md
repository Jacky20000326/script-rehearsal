# 使用者實機測試流程（v3）

> 給第一次拿到專案的人，從零跑完整個 v3「逐行真人錄音」流程。
> 含每個階段預期看到的 UI 反饋；建議用桌機 + Chrome（錄音可改用手機）。

## 0. 前置需求

- Node.js 20+ 與 npm
- 桌機 Chrome / Edge（其他現代瀏覽器亦可，Firefox 無 STT）
- 任一支援 MediaRecorder 的裝置做錄音（桌機 / 手機 / 平板皆可）
- 約 15 分鐘空檔

## 1. 安裝與啟動（< 2 分鐘）

```bash
cd "Script rehearsal"
npm install         # 約 30 秒（無 Whisper 依賴後變很快）
npm run dev         # 會自動先跑 npm run sync-script
# 開啟 http://localhost:3000
```

預期看到：
- 首頁標題「劇本對練」
- 角色選擇器（4 個按鈕：維克多 / 娜塔莉亞 / 胡利安 / 卡蘿莉娜）
- 範圍 / 提示模式選擇器
- 設定頁下半的「逐行錄音」區塊，每角色顯示進度條（`0 / 該角色總行數`）

## Step 1：v1.0 純 TTS 回歸（不錄音，< 3 分鐘）

驗證在「未錄任何音檔」狀態下，產品行為與 v1.0 完全一致。

1. 直接選角色「維克多」
2. 範圍選「第 41 頁」
3. 提示模式選「完整」
4. 點「開始對練」
5. 預期：所有對手台詞由瀏覽器 TTS 唸出；StatusBar 顯示來源「TTS」
6. 輪到維克多時系統暫停；念完按空白鍵或讓 STT 推進
7. 走完該頁 → 「練習完成 +1」彈窗

**通關條件**：StatusBar 從頭到尾都顯示「TTS」徽章，未顯示「真人錄音」。

## Step 2：錄音頁逐段錄、確認、跳行（5 分鐘）

1. 回首頁；找到「逐行錄音」區塊
2. 在「維克多」這列點「錄音」→ 跳轉到 `/record/維`（或 URL encoded）
3. 預期看到：
   - Header：「維克多 的錄音 ｜ 進度 0 / N」
   - 中央顯示第 1 行台詞（含 page / globalIndex 標籤）
   - 「⏺ 點擊開始錄音」按鈕
4. 第一次點時會跳出麥克風權限請求，允許
5. 點「⏺ 開始錄音」→ 唸完該行 → 點「⏹ 停止」
6. 進入 preview 狀態：可試聽、可「🔁 重錄」或「✓ 確認下一行」
7. 點「✓ 確認下一行」→ 預期：
   - Header 進度變 `1 / N`
   - 自動前進到第 2 行
   - 下方迷你地圖該位置打 ✓
8. 連續錄 3-4 行，驗證流程順暢
9. 中間嘗試點「◀ 上一行」/「下一行 ▶」測試跳轉
10. 在 preview 狀態下點跳行會跳 confirm「將丟棄這次錄音」

**通關條件**：可連續確認多行、跳行有 confirm 守衛、進度同步更新。

## Step 3：半錄 + 關頁 + 重開續錄（< 2 分鐘）

驗證 IndexedDB 持久化正確。

1. 在 Step 2 已錄 3-4 行的狀態下，直接關閉分頁（或按「返回設定」）
2. 重新開啟 `http://localhost:3000`
3. 設定頁「逐行錄音」區塊應顯示維克多進度 `3-4 / N`（與剛剛一致）
4. 再次點「錄音」進入 `/record/維`
5. 預期：
   - 已錄過的行在迷你地圖標 ✓
   - 游標停在第 1 行（可手動跳到下一未錄行繼續錄）

**通關條件**：關頁後計數正確、迷你地圖正確標記、可續錄。

## Step 4：全錄完 → 對練全程真人聲 + StatusBar 徽章（5 分鐘）

1. 把維克多剩下的行全錄完，進度跑到 `N / N`
2. 回首頁
3. **選非維克多的角色**（例如選「胡利安」自己念，對手就會有維克多的台詞）
4. 範圍選「第 41 頁」、提示模式選「完整」
5. 點「開始對練」
6. 走戲時注意：
   - 維克多的台詞會播放剛才錄的真人錄音
   - 其他未錄的角色（娜塔莉亞 / 卡蘿莉娜）仍走 TTS
   - StatusBar 在播放真人錄音時顯示「真人錄音」徽章；TTS 行顯示「TTS」徽章

**通關條件**：維克多每一行都播真人錄音、StatusBar 徽章正確切換。

## Step 5：重錄單行（< 1 分鐘）

1. 回首頁 → 點「錄音」進入維克多的錄音頁
2. 用迷你地圖點到任一已錄行（綠色 ✓）
3. 上方顯示「✓ 已錄」標記，但仍可重新錄
4. 點「⏺ 開始錄音」→ 重新錄 → 確認下一行
5. 對練時該行播放新版錄音

**通關條件**：可覆蓋舊片段；對練時播新版。

## Step 6：改 script.json → 橘色徽章 → 重錄缺漏行（3 分鐘）

驗證 scriptHash 機制。

1. 編輯**根目錄** `script.json`，改動維克多任何一行的內容（例如加個字）
2. 跑 `npm run sync-script`（或重啟 dev）
3. 回首頁
4. 預期「逐行錄音」區塊維克多列出現**橘色「劇本已變更」徽章**
5. 進對練頁仍可走，被動補的行（已錄但內容變動）依舊播放舊錄音
6. 進錄音頁，找到變動的那一行，重錄之
7. 全角色都重錄後，徽章消失（所有片段 hash 都對齊新劇本）

**通關條件**：橘色徽章正確出現；重錄完所有片段後徽章消失。

> 加碼驗證：若把所有維克多片段都刪光（Step 7），徽章也會消失（無錄音則無變更概念）。

## Step 7：刪除全部 → 回 v1.0（< 1 分鐘）

1. 設定頁 → 維克多列點「刪除全部」（或在角色卡操作）
2. 跳 confirm 「將刪除維克多所有錄音片段」→ 確認
3. 進度回到 `0 / N`、徽章消失
4. 開始對練 → 維克多的台詞全部走 TTS（StatusBar 顯示「TTS」徽章）

**通關條件**：刪除後完全回歸 v1.0 純 TTS 行為，無任何殘留。

## 通關條件總覽（給驗收人）

| # | 項目 |
|---|---|
| 1 | `npm install` + `npm run dev` 啟動無 error；package.json 已無 `@xenova/transformers` |
| 2 | Step 1 未錄音時對練全程 TTS，無「真人錄音」徽章 |
| 3 | Step 2 錄音頁可連續錄多行、preview 跳行有 confirm |
| 4 | Step 3 關頁重開計數與 ✓ 標記正確 |
| 5 | Step 4 全錄完對練時對應行播放真人錄音 + StatusBar 徽章正確 |
| 6 | Step 5 重錄覆蓋舊片段；對練播新版 |
| 7 | Step 6 改 script.json 後設定頁出現橘色徽章；全部重錄後徽章消失 |
| 8 | Step 7 刪除全部後回歸純 TTS |
| 9 | DevTools → IndexedDB → `script-rehearsal-audio` 只有 `audioSegments` 一個 store |
| 10 | 整段流程 console 無 error / warning（autoplay 提示可忽略） |

通過全部 10 項即代表 v3 在你的環境正常工作。
